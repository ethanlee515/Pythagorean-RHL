\section{Security Model and Checked Theorem}
\label{sec:security-model}

We first state the result without program-logic notation.  All distributions
are discrete.  Games return a Boolean, and
\(\WinProb[G]=\Pr[G=\mathsf{true}]\).  Statistical distance is
\(\TVD(P,Q)=\tfrac12\sum_x|P(x)-Q(x)|\), and \(\KL{P}{Q}\) denotes KL
divergence when the defining series is summable and \(P\) is absolutely
continuous with respect to \(Q\).

\subsection{Approximate FHE and noise flooding}

The formal scheme interface contains
\[
 S=(\mathsf{KeyGen},\mathsf{Enc},\mathsf{Eval}_1,
       \mathsf{Eval}_2,\mathsf{Dec}).
\]
Ciphertexts are either invalid or pairs \(\mathsf{Some}(c,e)\), where \(e\) is
an explicit error bound.  Unary and binary evaluation suffice to represent
circuits while keeping the package interface small.  Decryption is
distribution-valued in the generic interface, but the present proof assumes
it is concentrated at a pure function
\(\mathsf{dec}_0(sk,c)\).

To state flooding abstractly, messages carry a natural-valued metric \(d\)
and, for each center \(a\), maps
\[
 I_a:M\rightarrow\bbZ^n,
 \qquad J_a:\bbZ^n\rightarrow M.
\]
The theorem assumes
\begin{equation}
\label{eq:charts}
 I_a(a)=0,
 \quad d(a,b)=\norm{I_a(b)}_\infty,
 \quad J_b(v)=J_a(v+I_a(b)).
\end{equation}
The first two laws expose metric displacement as an integer vector; the last
says that changing the flooding center is translation in local coordinates.
These equations abstract the representative bookkeeping of lattice plaintext
spaces.  They are assumptions, not consequences of the generic FHE interface.

For \(\gamma>0\), the flooded transform
\(\mathsf{NF}_{\gamma}[S]\) keeps key generation, encryption, and evaluation
unchanged.  On \(c=\mathsf{Some}(\_,e)\), its decryption samples
\begin{align}
\label{eq:flood}
 m&\leftarrow\mathsf{Dec}(sk,c),&
 \sigma_e&=(e^2+1)\gamma,\notag\\
 \eta&\leftarrow D_{\bbZ^n,0,\sigma_e^2},&
 \mathsf{out}&=J_m(\eta+I_m(m)).
\end{align}
The tuple Gaussian has independent integer coordinates.  Invalid ciphertexts
produce the null subdistribution; the completed semantics used in the proof
make this missing mass explicit.

\subsection{IND-CPAD game}
\label{sec:ind-cpad}

IND-CPAD augments the usual left-or-right encryption game with homomorphic
evaluation and restricted decryption~\cite{lmss}.  The challenger samples a
hidden bit \(b\), generates \((pk,evk,sk)\), and gives \((pk,evk)\) to \(\cA\).
It maintains a table of triples \((m_0,m_1,c)\):

\begin{itemize}
\item \(\mathsf{OEnc}(m_0,m_1)\) encrypts \(m_b\) and appends the two labels
and resulting ciphertext;
\item \(\mathsf{OEval}\) evaluates indexed table ciphertexts and applies the
same gate to their left and right labels before appending a row;
\item \(\mathsf{ODec}(i)\) consumes one of \(q\) decryption credits.  It
returns no plaintext when the row's labels differ and otherwise returns the
decryption of its ciphertext.
\end{itemize}

The final result is whether \(\cA\)'s guess equals \(b\).  Thus decryption is
available only when it cannot trivially reveal which challenge message was
selected, but its approximation error may still depend on the secret key.
The full oracle pseudocode is in \Cref{app:ind-cpad-game}.

The Rocq adversary is a concrete stateful oracle program with the required
SSProve interface.  It may choose queries adaptively and may make arbitrary
numbers of encryption and evaluation calls.  The counter limits successful
entries into \(\mathsf{ODec}\) to \(q\); subsequent calls assertion-fail and
contribute failure mass.

\subsection{Simulation and reduction}

The IND-CPA reduction \(\cB_{\cA,q}\) runs \(\cA\), forwards pairs to its own
left-or-right encryption oracle, and reproduces evaluation and table updates
locally.  For a decrypt query on a row with equal labels \(m_0=m_1=m\), it
does not know \(sk\), so it samples a flooded answer centered directly at
\(m\).  When the labels differ it returns no plaintext, exactly as the real
game.  \Cref{alg:ind-cpa-reduction} gives executable-style pseudocode.

The lossy proof step compares the two centers
\(\mathsf{dec}_0(sk,c)\) and \(m\).  Support-level correctness gives
\[
 d(\mathsf{dec}_0(sk,c),m)\leq e.
\]
By \Cref{eq:charts}, every coordinate displacement has magnitude at most
\(e\).  The equal-variance Gaussian identity
\begin{equation}
\label{eq:dg-kl}
 \KL{\DG{\mu}{\sigma^2}}{\DG{\nu}{\sigma^2}}
   =\frac{(\nu-\mu)^2}{2\sigma^2}
\end{equation}
therefore bounds the \(n\)-coordinate KL cost of one replacement by
\begin{equation}
\label{eq:epsilon-nf}
 \epsilon_{\mathsf{nf}}
 \leq \frac{ne^2}{2(e^2+1)^2\gamma^2}
 \leq \frac{n}{2\gamma^2}.
\end{equation}

\subsection{The formal theorem boundary}

\caption{Inputs and obligations of the checked security theorem.  The first

computational interpretation.}

\toprule

\bottomrule
\end{tabular}
\end{table*}

\begin{theorem}[Checked noise-flooding reduction]
\label{thm:main}
Let \(S\), its metric charts, support-correctness proof, IND-CPA security
interface, and \(\gamma>0\) satisfy \Cref{tab:theorem-boundary}.  For every
valid IND-CPAD adversary package \(\cA\) and \(q\in\bbN\), the concrete package
\(\cB_{\cA,q}\) is a valid IND-CPA adversary and
\begin{align*}
 &\WinProb[
   \mathsf{IND\text{-}CPAD}_{\mathsf{NF}_{\gamma}[S],q}(\cA)]\\
 &\quad\leq \beta_{\mathsf{CPA}}(\cB_{\cA,q})
       +\sqrt{q\epsilon_{\mathsf{nf}}/2}\\
 &\quad\leq \beta_{\mathsf{CPA}}(\cB_{\cA,q})
       +\frac{\sqrt{qn}}{2\gamma}.
\end{align*}
\end{theorem}

The first inequality is the shape used internally.  In the artifact,
\(\epsilon_{\mathsf{nf}}=n/(2\gamma^2)\) is already the uniform bound from
\Cref{eq:epsilon-nf}, so the exported loss is written directly as
\(\sqrt{q(n/(2\gamma^2))/2}\).  The final theorem applies the assumed IND-CPA
bound to the checked reduction package; it does not assume IND-CPAD security.

For fixed \(n\) and target loss, \Cref{thm:main} requires
\(\gamma=\Theta(\sqrt q)\).  A statistical-distance hybrid using the same
one-call facts would require \(\gamma=\Theta(q)\).  Because \(\gamma\) scales
the noise actually returned by decryption, the improvement directly affects
precision rather than only proof aesthetics.
