\section{RHL with Pythagorean Errors}
\label{sec:program-logic}

We now describe the program logic used by the formalized noise-flooding proof.
The development does not introduce a new programming language.
Instead, it defines semantic judgments over SSProve \cite{ssprove} programs and proves the inference rules in Rocq.
The central design choice is to keep SSProve's concrete package/linking model, while adding two relational layers: an additive-error judgment for statistical distance arguments and a Pythagorean judgment for conditional KL-divergence bookkeeping.
The probability facts summarized in \Cref{sec:verified-kl-analysis} provide the distribution-level theorems consumed by these rules.

The rules in \Cref{fig:program-logic-core-rules} show the main proof principles used below.
Base rules are proved directly against the SSProve semantics or the distribution library.
Derived rules are still fully verified theorems, but their Rocq proofs mainly compose earlier judgment rules.
The compiler theorems are separated because they are not ordinary structural rules: their soundness rests on an explicit semantic induction over the trace compiler.

\subsection{Programs and Completed Semantics}

SSProve programs are written in the \texttt{raw\_code} monad shown in \Cref{fig:ssprove-monad}.
The constructors cover pure return, sampling, heap reads and writes, and calls to imported operations.
As usual in SSProve, cryptographic games are assembled by linking such code with packages.
This is important for the oracle-replacement theorem below: adversaries are not opaque objects, but concrete syntax that can be traversed and transformed.

\begin{figure}
\caption{SSProve command language used by the formalization}
\label{fig:ssprove-monad}
\begin{minted}{coq}
Inductive raw_code (A : choiceType) : Type :=
| ret (x : A)
| sampler (op : Op) (k : Arit op -> raw_code A)
| getr (l : Location) (k : l -> raw_code A)
| putr (l : Location) (v : l) (k : raw_code A)
| opr (o : opsig) (x : src o) (k : tgt o -> raw_code A).
\end{minted}
\end{figure}

The semantic function is SSProve's \texttt{Pr\_code}.
For an input $x$ and heap $m$, a program $c:X\rightarrow\code(Y)$ denotes a subdistribution
\[
    \semantic{c(x)}(m) \in \distr(Y\times\cM).
\]
Because SSProve semantics may be subprobabilistic, the judgments below use a completed distribution.

\begin{definition}[Completed distribution]
Let $\mu$ be a subdistribution over a set $A$.
The completed distribution $\completed{\mu}$ is the full distribution over $A\cup\{\bot\}$ defined by
\[
  \completed{\mu}(a)=\mu(a)
  \quad(a\in A),
  \qquad
  \completed{\mu}(\bot)=1-\sum_{a\in A}\mu(a).
\]
\end{definition}

For programs, we abbreviate $\completed{\semantic{c(x)}(m)}$ as $\csemantic{c(x)}(m)$.
For SSProve programs, output/heap pairs are first packed into a uniform choice type and then completed.
This explicit failure mass is what lets the rules account for failed traces and other loss of mass without an informal side condition.

No language extension is needed for discrete Gaussians: they are MathComp distributions embedded into SSProve sampling code by a small sampling wrapper.

\subsection{Judgments and Basic Rules}

\paragraph{Unary Hoare judgment.}
SSProve has exact relational Hoare logic.
For the Pythagorean rules, we use the corresponding one-sided support-style judgment:
\[
    \vDash\mathsf{Hoare}\set{P}\ c\ \set{Q}.
\]
It means that for every input and heap satisfying $P$, every element in the support of the SSProve semantics satisfies $Q$:
\[
\inferrule*[right=Hoare-Def]
  {
    \forall x,m.\ P(x,m)\Rightarrow
    \forall y,m'.\ (y,m')\in\supp\semantic{c(x)}(m)
      \Rightarrow Q(y,m')
  }
  {
    \vDash\mathsf{Hoare}\set{P}\ c\ \set{Q}
  }.
\]
It supplies the invariant-preservation facts for shared code, package calls, and trace residuals used by the Pythagorean sequencing rules.

\paragraph{Additive-error judgment.}
The completed additive-error judgment is
\[
    \vDash\AEJ_{\mathsf{opt}}\set{P}\ c_L \approx_\varepsilon c_R\ \set{Q}.
\]
Write $\mathsf{Cpl}(d;\mu,\nu)$ to mean that $d$ is a coupling of $\mu$ and $\nu$.
It means $0\leq\varepsilon$ and, for every related initial configuration, there is a coupling of the completed output distributions that satisfies $Q$ with probability at least $1-\varepsilon$:
\[
\begin{aligned}
&\vDash\AEJ_{\mathsf{opt}}\set{P}\ c_L \approx_\varepsilon c_R\ \set{Q}
\\
&\quad\Longleftrightarrow\quad
0\leq\varepsilon\ \wedge\
\forall x_L,x_R,m_L,m_R.\ P((x_L,m_L),(x_R,m_R))\Rightarrow
\\
&\qquad\exists d.\ 
\mathsf{Cpl}\!\left(d;\csemantic{c_L(x_L)}(m_L),
\csemantic{c_R(x_R)}(m_R)\right)
\ \wedge\ \Pr_{d}[Q]\geq 1-\varepsilon .
\end{aligned}
\]
The ordinary judgment $\vDash\AEJ$ is a specialization where $Q$ is a postcondition on successful output/heap pairs, lifted through completion.
We use $\vDash\AEJ_{\mathsf{opt}}$ as the primary judgment because the explicit failure point is needed for trace compilation and completed-output TVD bounds.
\Cref{fig:program-logic-core-rules} shows the two main ways to prove an additive-error judgment.
\LabTirName{AE-Couple} constructs the coupling directly, while \LabTirName{AE-Completed-TVD-Eq} derives the judgment from a TVD bound on completed outputs.
The formalization also includes consequence, sequencing, and total-output variants.

Compared with EasyCrypt, this is a semantic pRHL/up-to-bad layer with the bad event already internalized into the coupling success probability.
Compared with SSProve out of the box, we reuse SSProve's distributions, couplings, heaps, packages, and linking semantics, but add a reusable approximate judgment over program fragments.

\paragraph{Pythagorean judgment.}
The Pythagorean judgment is
\[
    \vDash\Pyth\set{P}\ c_L \approx_s c_R\ \set{Q},
\]
where $s$ is a tuple of nonnegative KL budgets.
For every related initial configuration, the judgment existentially produces two trace distributions $\cP,\cQ$ over tuples of completed output/heap values.
The final coordinate of $\cP$ and $\cQ$ must marginalize to the completed semantics of $c_L$ and $c_R$, and every conditional coordinate distribution must satisfy the KL bound recorded by $s$.
The postcondition $Q$ is checked on the successful support of each concrete program output.

Rather than merely listing syntactic sampling sites, the judgment records a full transcript distribution over completed outputs.
That transcript is then handed to the probability-level theorem.
The underlying distribution predicate is the Pythagorean distribution predicate described in \Cref{sec:verified-kl-analysis}.

\subsection{Pythagorean Error Budgets}

The mathematical bridge from conditional KL budgets to TVD is
\Cref{lem:verified-pythagorean-preservation}.
The program logic uses that lemma through a final-marginal TVD theorem: once a Pythagorean judgment has produced transcript distributions satisfying the Pythagorean distribution predicate, the final completed-output marginals are within
\[
  \sqrt{\frac{\sum_i s_i}{2}}
\]
in total variation.
For a Pythagorean budget tuple $s$, we write
\[
  \lVert s\rVert_1 \triangleq \sum_i s_i .
\]
This is the ordinary $\ell_1$ norm because the judgment requires every coordinate of $s$ to be nonnegative.

For the compiler theorem, we use a call-budget tuple $\operatorname{callErr}_q(\varepsilon)$ defined by
\[
\begin{aligned}
\operatorname{callErr}_0(\varepsilon)
  &= (0,\varepsilon,0),\\
\operatorname{callErr}_{q+1}(\varepsilon)
  &= (0,\varepsilon)\mathbin{+\!\!+}
     \operatorname{callErr}_q(\varepsilon).
\end{aligned}
\]
Thus $\operatorname{callErr}_q(\varepsilon)$ contains $q+1$ copies of $\varepsilon$ separated by zero-budget bookkeeping steps, and its TVD bookkeeping lemma is:
\[
    \operatorname{pythTV}
      (\operatorname{callErr}_q(\varepsilon))
    =
    \sqrt{\frac{(q+1)\varepsilon}{2}}.
\]
The oracle-replacement theorem below phrases the bound with a direct call budget $q$, hence $\sqrt{q\varepsilon/2}$.

\subsection{Pythagorean Proof Rules}

\paragraph{Sampling rules.}
The primitive sampling rule embeds an ordinary distribution-level KL fact into a one-coordinate Pythagorean judgment:
\begin{mathpar}
\inferrule[KL-Samp]
  {
    0\leq\varepsilon\\
    \forall x_L,x_R,m_L,m_R.\
      P((x_L,m_L),(x_R,m_R))\Rightarrow m_L=m_R\\
    \forall x_L,x_R.\ \mathsf{finiteKL}(D_L(x_L),D_R(x_R))\\
    \forall x_L.\ dweight(D_L(x_L))=1\\
    \forall x_R.\ dweight(D_R(x_R))=1\\
    \forall x_L,x_R,m_L,m_R.\
      P((x_L,m_L),(x_R,m_R))
      \Rightarrow \KL{D_L(x_L)}{D_R(x_R)}\leq\varepsilon\\
    \forall x_L,x_R,m_L,m_R,y.\
      P((x_L,m_L),(x_R,m_R))\wedge y\in\supp(D_L(x_L))
      \Rightarrow Q(y,m_L)\\
    \forall x_L,x_R,m_L,m_R,y.\
      P((x_L,m_L),(x_R,m_R))\wedge y\in\supp(D_R(x_R))
      \Rightarrow Q(y,m_R)
  }
  {
    \vDash\Pyth\set{P}\
      \mathsf{sample}(D_L)
      \approx_{[\varepsilon]}
      \mathsf{sample}(D_R)
      \set{Q}
  }.
\end{mathpar}
The equal-memory premise is present because sampling returns the sampled value without changing the heap, and the one-step trace encoding packs output/heap pairs.

The discrete Gaussian rule is the corresponding specialization:
\begin{mathpar}
\inferrule[KL-DG]
  {
    0<\sigma\\
    0\leq\varepsilon\\
    \forall \mu_L,\mu_R,m_L,m_R.\
      P((\mu_L,m_L),(\mu_R,m_R))\Rightarrow m_L=m_R\\
    \forall \mu_L,\mu_R,m_L,m_R.\
      P((\mu_L,m_L),(\mu_R,m_R))\Rightarrow
      \frac{(\mu_R-\mu_L)^2}{2\sigma^2}\leq\varepsilon\\
    \forall \mu_L,\mu_R,m_L,m_R,y.\
      P((\mu_L,m_L),(\mu_R,m_R))
      \wedge y\in\supp(\mathcal{DG}_{\mu_L,\sigma^2})
      \Rightarrow Q(y,m_L)\\
    \forall \mu_L,\mu_R,m_L,m_R,y.\
      P((\mu_L,m_L),(\mu_R,m_R))
      \wedge y\in\supp(\mathcal{DG}_{\mu_R,\sigma^2})
      \Rightarrow Q(y,m_R)
  }
  {
    \vDash\Pyth\set{P}\
      x\gets\mathcal{DG}_{\mu_L,\sigma^2}
      \approx_{[\varepsilon]}
      x\gets\mathcal{DG}_{\mu_R,\sigma^2}
      \set{Q}
  }.
\end{mathpar}
The formal rule uses integer centers and the verified KL bound for discrete Gaussians.
There is also a closed-program variant.

\paragraph{Structural rules.}
The reflexivity and consequence rules are proved directly from the transcript semantics:
\begin{mathpar}
\inferrule[Pyth-Refl]
  {
    \forall i.\ 0\leq s_i\\
    \forall x_L,x_R,m_L,m_R.\
      P((x_L,m_L),(x_R,m_R))\Rightarrow x_L=x_R\wedge m_L=m_R\\
    \forall x,m,y,m'.\
      P((x,m),(x,m))\wedge (y,m')\in\supp\semantic{c(x)}(m)
      \Rightarrow Q(y,m')
  }
  {
    \vDash\Pyth\set{P}\ c\approx_s c\ \set{Q}
  }

\inferrule[Pyth-Conseq]
  {
    P'\Rightarrow P\\
    Q\Rightarrow Q'\\
    \forall i.\ s_i\leq s'_i\\
    \vDash\Pyth\set{P}\ c_L\approx_s c_R\ \set{Q}
  }
  {
    \vDash\Pyth\set{P'}\ c_L\approx_{s'} c_R\ \set{Q'}
  }
\end{mathpar}
Notice that \LabTirName{Pyth-Refl} permits any nonnegative budget tuple.
This is just monotonicity of the KL bounds: identical traces have zero KL, so they fit inside any nonnegative budget.

The main sequencing rule concatenates error tuples:
\begin{mathpar}
\inferrule[Pyth-Seq]
  {
    \vDash\Pyth\set{P}\ c_L\approx_{s_1} c_R\ \set{M}\\
    \vDash\Pyth
      \set{y_L=y_R\wedge m_L=m_R\wedge M(y_L,m_L)}
      \ k_L \approx_{s_2} k_R\ \set{Q}
  }
  {
    \vDash\Pyth\set{P}\
      (y\gets c_L;\ k_L(y))
      \approx_{s_1\mathbin{+\!\!+}s_2}
      (y\gets c_R;\ k_R(y))
      \set{Q}
  }.
\end{mathpar}
The second premise is written pointwise: for every intermediate value and heap satisfying $M$, the two continuations are related when run from the same value and heap.
This pointwise formulation avoids a separate syntactic rule for every command constructor.

The mixed Hoare/Pythagorean sequencing rules let the proof run shared code without spending a KL budget.
They are used repeatedly in the compiler proof:
\begin{mathpar}
\inferrule[Pyth-Hoare-Seq]
  {
    P((x_L,m_L),(x_R,m_R))
      \Rightarrow x_L=x_R\wedge m_L=m_R\wedge P_0(x_L,m_L)\\
    \vDash\mathsf{Hoare}\set{P_0}\ c\ \set{M}\\
    \vDash\Pyth
      \set{y_L=y_R\wedge m_L=m_R\wedge M(y_L,m_L)}
      \ k_L\approx_s k_R\ \set{Q}
  }
  {
    \vDash\Pyth\set{P}\
      (y\gets c;\ k_L(y))
      \approx_{[0]\mathbin{+\!\!+}s}
      (y\gets c;\ k_R(y))
      \set{Q}
  }

\inferrule[Pyth-Shared-Cont]
  {
    \vDash\Pyth\set{P}\ c_L\approx_s c_R\ \set{M}\\
    \vDash\mathsf{Hoare}\set{M}\ k\ \set{Q}
  }
  {
    \vDash\Pyth\set{P}\
      (y\gets c_L;\ k(y))
      \approx_{s\mathbin{+\!\!+}[0]}
      (y\gets c_R;\ k(y))
      \set{Q}
  }
\end{mathpar}
There are closed-program variants for hiding unit inputs.

\paragraph{Micciancio-Walter bridge.}
The rule that turns a Pythagorean judgment into an additive-error judgment is:
\begin{mathpar}
\inferrule[Micciancio-Walter]
  {
    \vDash\Pyth\set{P}\ c_L\approx_s c_R\ \set{Q}\\
    \delta=\sqrt{\frac{\sum_i s_i}{2}}
  }
  {
    \vDash\AEJ_{\mathsf{opt}}\set{P}\ c_L
      \approx_\delta c_R\ \set{o_L=o_R}
  }.
\end{mathpar}
The conclusion is intentionally about equality of completed outputs.
The postcondition $Q$ is part of the Pythagorean judgment and is used by sequencing rules, but the bridge theorem extracts the statistical-distance/equality consequence needed for cryptographic indistinguishability.

This is the point where the formalization realizes Pythagorean probability preservation: the final-marginal TVD theorem bounds the TVD of completed outputs, and \LabTirName{AE-Completed-TVD-Eq} turns that TVD bound into the completed additive-error judgment.

\subsection{Trace Compilation and Oracle Replacement}

The oracle-replacement story is not an EasyCrypt-style rule for opaque adversaries.
Instead, the formalization defines an explicit compilation pass over concrete SSProve code.

\begin{definition}[Trace compiler]
Fix an operation identifier $fn$ and a package $P$ implementing that operation.
A trace $\tau$ is a finite list of effect entries
\[
    e ::= \mathsf{call}(y)\mid \mathsf{get}(v)\mid \mathsf{put}\mid \mathsf{sample}(a).
\]
For a program $c$, define the partial residual operation
$\operatorname{cont}(c,\tau)$ by replaying $\tau$ against $c$:
\[
\begin{array}{rcl}
\operatorname{cont}(c,[\,])&=&c,\\
\operatorname{cont}(\mathsf{ret}(v),e::\tau)&=&\bot,\\
\operatorname{cont}(\mathsf{opr}(o,x,k),\mathsf{call}(y)::\tau)
  &=&\operatorname{cont}(k(y),\tau),\\
\operatorname{cont}(\mathsf{get}(l,k),\mathsf{get}(v)::\tau)
  &=&\operatorname{cont}(k(v),\tau),\\
\operatorname{cont}(\mathsf{put}(l,v,k),\mathsf{put}::\tau)
  &=&\operatorname{cont}(k,\tau),\\
\operatorname{cont}(\mathsf{sample}(D,k),\mathsf{sample}(a)::\tau)
  &=&\operatorname{cont}(k(a),\tau),
\end{array}
\]
with result $\bot$ when the next trace entry has the wrong tag or payload type.

Define $\operatorname{runToCall}(c,fn,\tau)$ recursively by executing $c$ until it either returns or reaches the next call whose identifier is $fn$:
\[
\begin{array}{rcl}
\operatorname{runToCall}(\mathsf{ret}(v),fn,\tau)
  &=& \mathsf{ret}(\mathsf{done}(v),\tau),\\
\operatorname{runToCall}(\mathsf{opr}((fn,S,T),x,k),fn,\tau)
  &=& \mathsf{ret}(\mathsf{query}(x),\tau),\\
\operatorname{runToCall}(\mathsf{opr}(o,x,k),fn,\tau)
  &=& y\gets o(x);\ 
      \operatorname{runToCall}(k(y),fn,\tau\cdot\mathsf{call}(y)),\\
\operatorname{runToCall}(\mathsf{get}(l,k),fn,\tau)
  &=& v\gets \mathsf{get}(l);\ 
      \operatorname{runToCall}(k(v),fn,\tau\cdot\mathsf{get}(v)),\\
\operatorname{runToCall}(\mathsf{put}(l,v,k),fn,\tau)
  &=& \mathsf{put}(l,v);\ 
      \operatorname{runToCall}(k,fn,\tau\cdot\mathsf{put}),\\
\operatorname{runToCall}(\mathsf{sample}(D,k),fn,\tau)
  &=& a\gets D;\ 
      \operatorname{runToCall}(k(a),fn,\tau\cdot\mathsf{sample}(a)).
\end{array}
\]
Now define the trace-producing helper $\operatorname{factorCalls}_q(P,fn,c,\tau)$ by recursion on $q$:
\[
\begin{array}{rcl}
\operatorname{factorCalls}_0(P,fn,c,\tau)
  &=&\mathsf{ret}(\tau),\\
\operatorname{factorCalls}_{q+1}(P,fn,c,\tau)
  &=&
  \begin{cases}
  \mathsf{bad}, & \operatorname{cont}(c,\tau)=\bot,\\
  s\gets\operatorname{runToCall}(c',fn,[\,]);\ F_q(s),
    & \operatorname{cont}(c,\tau)=c',
  \end{cases}
\end{array}
\]
where
\[
\begin{array}{rcl}
F_q(\mathsf{done}(v),\tau')&=&\mathsf{ret}(\tau\cdot\tau'),\\
F_q(\mathsf{query}(x),\tau')
  &=& y\gets \mathsf{resolve}(P,fn,x);\ 
      \operatorname{factorCalls}_q
        (P,fn,c,\tau\cdot\tau'\cdot\mathsf{call}(y)).
\end{array}
\]
Finally, the compiled program is
\[
    \operatorname{compileCalls}_q(P,fn,c)
    =
    \tau\gets\operatorname{factorCalls}_q(P,fn,c,[\,]);\
    \begin{cases}
      c',& \operatorname{cont}(c,\tau)=c',\\
      \mathsf{bad},& \operatorname{cont}(c,\tau)=\bot.
    \end{cases}
\]
\end{definition}

The formal definitions pack traces and call inputs into SSProve choice types, but the mathematical control flow is the one above.

The same-package correctness theorem is:
\begin{mathpar}
\inferrule[Compile-Same-Package]
  {
    \mathsf{ValidPackage}_{L_P,\emptyset,M}(P)\\
    \mathsf{fhas}(M,fn:X\to Y)\\
    \mathsf{ValidCode}_{L,M}(c)
  }
  {
    \mathsf{codeLink}(
      \operatorname{compileCalls}_q(P,fn,c),P)
    =
    \mathsf{codeLink}(c,P)
  }.
\end{mathpar}
Thus the compiler is not an additional cryptographic assumption; linked with the same package, it is semantics-preserving.

The formalization proves the quantitative replacement theorem for this compiled form.
Write $\mathcal{O}_{P'}(x)$ for $\mathsf{resolve}(P',fn,x)$, and write
$A_{P'}^{(q)}$ for the linked adversary obtained by compiling the first $q$
selected calls through $P'$.
The local assumption is a one-call Pythagorean judgment for the selected operation.
The public replacement rule is stated in its tuple-budget form:
\[
    \vDash\Pyth
      \set{x_L=x_R\wedge m_L=m_R\wedge I(m_L)}
      \mathcal{O}_{P'} \approx_s \mathcal{O}_{P''}
      \set{I(m')}.
\]
The singleton tuple $s=(\varepsilon)$ recovers the scalar one-call rule.
The displayed rules are schematic only in the SSProve packaging parameters.
First define the heap-region and package-preservation predicates
\[
\begin{aligned}
m_0\equiv_K m_1
&\Longleftrightarrow
\forall \ell.\ \ell\in K\Rightarrow m_0[\ell]=m_1[\ell],
\\
\mathsf{DependsOnlyOn}_K(I)
&\Longleftrightarrow
\forall m_0,m_1.\ m_0\equiv_K m_1
  \Rightarrow (I(m_0)\Leftrightarrow I(m_1)),
\\
\mathsf{PresExcept}_{M,P',fn}(I)
&\Longleftrightarrow
\forall o\in M.\ \mathsf{id}(o)\neq fn\Rightarrow
\\
&\qquad
\vDash\mathsf{Hoare}
  \set{(u,m)\mapsto I(m)}
\\
&\qquad\quad
  (u\mapsto\mathsf{resolve}(P',o,u))
\\
&\qquad\quad
  \set{(v,m')\mapsto I(m')}.
\end{aligned}
\]
Let $\mathcal{H}_{\mathsf{cmp}}$ abbreviate the following conjunction:
\[
\begin{aligned}
\mathcal{H}_{\mathsf{cmp}}
\Longleftrightarrow{}&
(\forall x.\ \mathsf{ValidCode}_{L,M}(\mathsf{prog}(x)))\\
&{}\wedge
\mathsf{ValidPackage}_{L',\emptyset,M}(P')
\wedge
\mathsf{ValidPackage}_{L'',\emptyset,M}(P'')\\
&{}\wedge
\mathsf{fseparate}(K,L)
\wedge
\mathsf{fhas}(M,fn:X\to Y)\\
&{}\wedge
\mathsf{DependsOnlyOn}_K(I)
\wedge
\mathsf{PresExcept}_{M,P',fn}(I).
\end{aligned}
\]
The fully quantified theorem keeps the SSProve location, interface, and package-validity parameters explicit.

For readability, the intermediate Pythagorean compiler rule is shown as a
scalar instance of the tuple-budget theorem used by
\LabTirName{Compile-Replace}.
\begin{mathpar}
\inferrule[Pyth-Compile-Calls]
  {
    \mathcal{H}_{\mathsf{cmp}}\\
    \vDash\Pyth_1\set{P_I}\ \mathcal{O}_{P'}
      \approx_\varepsilon \mathcal{O}_{P''}\ \set{I(m')}
  }
  {
    \vDash\Pyth\set{P_I}
      \ A_{P'}^{(q+1)}
      \approx_{\operatorname{callErr}_q(\varepsilon)}
      \ A_{P''}^{(q+1)}
      \set{I(m')}
  }.
\end{mathpar}
Here $P_I$ abbreviates $x_L=x_R\wedge m_L=m_R\wedge I(m_L)$.
The proof is by induction over the call bound.
Each inductive step spends one $\varepsilon$ entry for the selected package call and uses Hoare rules for the shared search/resume code around it.
This compiler family is the most involved program-logic proof in the development: it combines the call-bound induction with residual/replay correctness, invalid-trace cases, heap-region preservation, and tuple-budget bookkeeping.

Composing this with \LabTirName{Micciancio-Walter} yields the oracle-replacement rule:
\begin{mathpar}
\inferrule[Compile-Replace]
  {
    \mathcal{H}_{\mathsf{cmp}}\\
    \vDash\Pyth\set{P_I}\ \mathcal{O}_{P'}
      \approx_s \mathcal{O}_{P''}\ \set{I(m')}
  }
  {
    \vDash\AEJ_{\mathsf{opt}}
      \set{P_I}
      \ A_{P'}^{(q)}
      \approx_{\sqrt{q\lVert s\rVert_1/2}}
      \ A_{P''}^{(q)}
      \set{o_L=o_R}
  }.
\end{mathpar}
This tuple form is the one used by the noise-flooding proof: the selected decryption body is factored into a shared deterministic prefix, one discrete-Gaussian sampling coordinate, and a shared deterministic continuation.
Informally, this is the oracle rule
\[
    O_i\sim_s O_i'
    \quad\Longrightarrow\quad
    A^{O_1,\ldots,O_i,\ldots}
    \sim_{\sqrt{q\lVert s\rVert_1/2}}
    A^{O_1,\ldots,O_i',\ldots}
\]
for concrete SSProve adversaries after compiling the first $q$ calls to the selected operation.

Compared with EasyCrypt, the difference is mostly where the abstraction boundary sits.
EasyCrypt's adversarial call rules reason about opaque adversary modules.
Here the adversary is executable syntax, so the proof factors the adversary by a verified trace compiler.
Compared with stock SSProve, the same-package theorem is ordinary package-linking correctness, while \LabTirName{Compile-Replace} adds the quantitative statement that replacing one selected operation by a KL-close implementation changes the compiled adversary's completed output distribution by at most $\sqrt{q\lVert s\rVert_1/2}$.
